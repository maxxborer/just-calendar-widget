# Security Policy

## Supported versions

Security fixes are provided for the latest release on the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's **Report a vulnerability** button in the repository's **Security** tab when it is available. If private reporting is not enabled, email [maxxborer@gmail.com](mailto:maxxborer@gmail.com) with a concise reproduction, affected version, and potential impact.

You will receive an acknowledgement within seven days. Please allow time for a fix and coordinated disclosure before sharing details publicly.
